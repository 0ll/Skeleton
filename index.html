Skeleton Sass

Skeleton Sass is a sass-based transcription of Dave Gamache's Skeleton CSS.

Faster

Skeleton Sass has been cut in size by 1000% and is faster than ever before!

Customizable

Customize Skeleton Sass without fear of losing all your hard work!

Simple

Skeleton Sass 2 is still the same, it's just a little easier to use!

  Free!

Skeleton Sass is completely free to use. Check out the license.

  No JavaScript

Skeleton Sass has no dependency on JavaScript. Skeleton Sass is built with Sass which compiles down to vanilla CSS. Learn more about sass.

  Responsive

Skeleton Sass scales to your device whether it is a desktop computer, laptop, tablet, or mobile phone

  Github

Skeleton Sass is on Github! Help us out and fork the project and be social!

  Customize

Skeleton Sass is easily customizable to suit your project needs with friendly mixins, useful functions, configuration overrides, and much more!

  CSS3

Skeleton Sass 2 adds some necessary CSS3 features such as global border-box and many other mixins that make dealing with CSS3 features easy and fun again.

Requirements

Skeleton Sass 2 has only a single requirement: Sass 3.3. If you wish to integrate other Sass libraries into Skeleton Sass, just simply add it as an import in skeleton_template.scss or bundle it with your theme to start using it!

Structure

Skeleton Sass 2 has an intuitive file structure that allows for greater customization. Note: if you are a Skeleton Sass 1 user, the structure for Skeleton Sass 2 is vastly different.

  • bin/ – this directory contains executables (shell scripts) that perform various tasks
    • setup.sh – automates setting up Skeleton Sass 2 for first use!
    • theme_setup.sh – automates setting up a brand new theme
    • update.sh – automates updating the script via git, bower, or sache (script determines which method to use)
    • upgrade – aids in successfully upgrading from Skeleton Sass 1 to Skeleton Sass 2
  • skeleton/ – this directory contains the heart and personality of the framework
    • core/ – this entire directory contains the core of the framework
      • _config.scss – this file contains all of the globally default configuration variables and values
      • _dependencies.scss – this file contains all logic that goes into creating the public-facing functions and mixins
      • _functions.scss – this file contains all the globally public functions
      • _mixins.scss – this file contains all of the globally public mixins
    • themes/ – this directory contains the personality of the framework. Add new themes here so your changes don't get overridden with an update!
      • demo/ – a simple demo theme for you to base your theme off of
        • _base.scss – all the base styles for your new theme
        • _vars.scss – all project-scoped mixins, functions, etc. go in here
      • sphenoid/ – the default look of Skeleton Sass that's still identical to Skeleton CSS
        • marrow/ – a directory that holds project-scoped logic, functions, mixins, and other such constructs
          • _private.scss – contains all the project-scoped logic
          • _public.scss – contains all the project-scoped front-facing mixins, functions, and other such constructs
        • _base.scss – all the base styles to make sphenoid look like a boss
        • _skeleton.scss – all the styles that generate the grid
        • _vars.scss – all project-scoped overrides and new settings that should not be global go here
      • _loader.scss – a simple file that loads the currently selected theme
  • _MYconfig.scss – your file for global overrides, just rename and your changes will be preserved on update!
  • skeleton_template.scss the file that accumulates all of the styles together in a simple and cohesive file. This is file is where all of the styles for your project can go if you wish. Simply rename to skeleton.scss and compile for the first time and your changes will be preserved after an update!

Resources

Below are a few resources for you to learn more about Sass, Bower, Sache, Sass compilers, and other Sass libraries:

Learn Sass
  • You can learn a lot about Sass even if you're brand new to web development! Check out the official Sass website
  • Another excellent free resource for learn more about Sass is thesassway.com
Compile Sass
  • Command line via sass command
    • OS X – thankfully, most OS X platforms come bundled with Ruby and Ruby Gems so the setup will be minimal! Learn more about how to install Ruby and Sass here
    • Windows – it is just as easy for Windows users as OS X users. Simply install ruby via Ruby Installer (be sure to add Ruby to your PATH) and then resume with the installation here
    • Linux – If you're using Linux it might be fair to assume that you have experience with terminal. If your Linux installation does not come with ruby, be sure to install it (either from source or from your package manager) and then resume with the installation here
  • Applications
    • CodeKit (paid)
    • Compass.app (paid, open source)
    • Hammer (paid)
    • Koala (free, open source)
    • LiveReload (paid, open source)
    • Mixture (paid)
    • Prepros (paid, open source)
    • Scout (free, open source)
Extend Sass

There are plenty of ruby gems, libraries, and frameworks that extend Sass but we'll just go over two ways to easily extend the capabilities of Sass.

  • Bower.io – Bower is an easy way to gather dependencies for any given frontend software that exists in bower's list. Skeleton Sass is among many softwares in that list including jQuery and Linux Kernal.
  • Sache.in – Sache is a utility created exclusively for Sass and Compass projects. It's an excellent resource for extending Sass!
